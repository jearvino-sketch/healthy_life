# 江浙探店评论区饭店实体提取报告

核验日期：2026-07-22。

## 结果摘要

- 视频发现范围已扩展到无锡、苏州、常州、镇江及其县级市，共有 751 条视频通过“粉丝数大于 5,000、评论数大于 20”及语义门槛。
- 新来源的评论重新采集在首条请求返回 HTTP 412 后停止；因此本报告的实体结果仍以此前无锡来源中当前公开可见并去重的 28,505 条历史评论为语料，不能代表四市 751 条视频的最终评论实体分布。
- 本轮可实际逐条扫描 28,207 条：35 份原始 JSON 中的 28,195 条全文，加上两条历史视频保留下来的 12 条精选证据摘要；覆盖率 98.95%。
- 余下 298 条历史原文当前不可恢复：两条旧视频重新请求均返回 HTTP 412；内置浏览器能读取主评论，但“展开回复”持续超时。结果不把这 298 条冒充为已重新扫描。
- 第一阶段宽松召回并按候选写法去重后得到 2,005 条；这一层只负责找可能的名称，不负责证明推荐成立。
- 第二阶段按别名归一为 145 个实体，并对每个“实体—评论”组合独立判断语义，共输出 1,878 条标注：307 条是有效推荐，1,571 条是戏谑／比较、负面或仅提及。84 个实体至少有一条有效推荐，61 个没有明确推荐。
- 145 个实体中，11 个已有高德与点评双平台记录，5 个仅有地图记录，2 个仅有点评记录，127 个仍待平台核验。24 个短名或同名较多的实体标记为“歧义待解”。
- “淘米水”在现有三个相关视频中的 9 条提及经人工复核均属于同一反讽梗语境，包括字面写成“确实也不错”“喜欢吃”的评论；全部记为“戏谑或比较”，有效推荐数为 0。“京东食堂／京东外卖”属于戏谑对象或服务名，不进入饭店候选。

## 两阶段文件

1. [饭店提及候选.csv](饭店数据/饭店提及候选.csv)是高召回表。它故意保留缩写、错写和部分噪声；不能把每一行直接当成饭店或推荐。
2. [饭店实体精筛.csv](饭店数据/饭店实体精筛.csv)是归一实体表。只有进入[饭店别名词典.csv](饭店数据/饭店别名词典.csv)的名称才会出现，并分别列出有效推荐、戏谑／比较、负面和仅提及语境数。
3. [饭店推荐语义标注.csv](饭店数据/饭店推荐语义标注.csv)保存每条实体评论的原文、语义角色、是否有效推荐及判定依据，是实体汇总数字的可追溯明细；[饭店有效推荐评论.csv](饭店数据/饭店有效推荐评论.csv)只保留其中 307 条有效推荐，便于直接复核。
4. [饭店语义规则.csv](饭店数据/饭店语义规则.csv)保存经人工确认的视频级语境例外；规则限定到具体 BV，避免把其他视频里同名饭店的真实评价一并排除。
5. [门店核验.csv](饭店数据/门店核验.csv)保存具体分店、地址和平台链接。品牌命中但评论未指明分店时，不把不同分店强行合并。

两张实体生成表新增“涉及城市”和“涉及区县或县级市”字段，按 BV 号回连[视频来源.csv](饭店数据/视频来源.csv)。历史视频若已不在当前严格来源表中，这两个字段保持空白，不通过店名或文本猜测地区。

“评论提及数”只是包含该名称或别名的去重评论数；只有语义标注明确为“有效推荐”的评论才进入有效推荐计数。实体仍需完成分店和平台核验，不能只凭评论数进入 A／B／C 正式证据等级。

## 召回规则

- 词典命中：对已知正式名、常见简称、大小写和疑似错写进行统一，例如 `PUNCH`／`Punch House`、`鸡鸭鹅大饭店`／`无锡鸡鸭鹅大酒店`。
- 名称召回与推荐判断分离：正则和词典只用于发现可能的店名；推荐判断读取包含该实体的完整分句，不把整条评论中的正负词复制给所有店名。
- 比较／戏谑：`不如 X`、`还不如 X`、反问、表情和已人工确认的评论梗只算提及，不自动推导为对 X 的推荐。
- 有效推荐：需要出现针对该实体的明确到店／菜品建议、正向用餐体验、重复到店行为或处于明确推荐清单；负面、反讽和纯名称罗列单独计数。
- 门店后缀：召回以“面馆、饭店、食府、馄饨店、小笼、包子、烤肉、酸辣汤”等结尾的片段。
- 去重：先按 `BV号 + 评论ID` 去重评论，再按候选写法聚合；精筛表进一步按标准店名合并别名。
- 隐私最小化：输出不含评论者昵称、UID、头像或主页，只保留 BV 号、评论 ID 和最小必要证据摘录。

## 复跑

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 饮食\脚本\提取饭店实体.ps1
```

默认读取 `%TEMP%\healthy_life_bili_comments\*.json`。脚本会重建两张输出 CSV；别名判断应改在词典中，平台信息应改在门店核验表中，避免手工修改生成表后被覆盖。

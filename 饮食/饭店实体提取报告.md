# 江浙探店评论区饭店实体提取报告

核验日期：2026-07-22。

## 结果摘要

- 视频发现范围已扩展到无锡、苏州、常州、镇江及其县级市，共有 751 条视频通过“粉丝数大于 5,000、评论数大于 20”及语义门槛。
- 751 条正式来源已全部完成低速、可断点重采集，实际取得 191,704 条评论；此前的 HTTP 412 网络阻塞未再出现，状态表中没有失败项。535 条视频达到页面展示评论总数，216 条因页面计数变化、删除或接口可见性差异标记为未完整。
- 本轮实际逐条扫描 191,705 条记录：751 份正式原始 JSON 中的 191,704 条全文，加上当前原始文件缺失的 1 条历史精选证据摘要。临时目录里另有 11 份早期试抓 JSON，不属于本批正式 BV 清单，已由脚本排除。
- 第一阶段宽松召回并按候选写法去重后得到 12,426 条；这一层只负责找可能的名称，不负责证明推荐成立。
- 第二阶段按别名词典和候选人工审计归一为 312 个实体，并对每个“实体—评论”组合独立判断语义，共输出 6,919 条标注：1,468 条是有效推荐，5,451 条是戏谑／比较、负面或仅提及。269 个实体至少有一条有效推荐，43 个没有明确推荐。
- 312 个实体中，11 个已有高德与点评双平台记录，7 个仅有地图记录，3 个仅有点评记录，291 个仍待平台核验。短名、同名较多或跨城市出现的实体继续标记为“歧义待解”。
- “淘米水”已确认对应真实饭店“淘米水私房菜（向阳路店）”，因此撤销此前按三个视频整体强制标为反讽的规则；`不如淘米水` 等比较句仍不计推荐，而明确的正向体验按评论自身语义计数。“京东食堂／京东外卖”仍属于服务名，不进入饭店候选。

## 两阶段文件

1. [饭店提及候选.csv](饭店数据/饭店提及候选.csv)是高召回表。它故意保留缩写、错写和部分噪声；不能把每一行直接当成饭店或推荐。
2. [饭店实体精筛.csv](饭店数据/饭店实体精筛.csv)是归一实体表。进入[饭店别名词典.csv](饭店数据/饭店别名词典.csv)或经[饭店候选人工审计.csv](饭店数据/饭店候选人工审计.csv)确认新增／合并的名称会出现，并分别列出有效推荐、戏谑／比较、负面和仅提及语境数。
3. [饭店推荐语义标注.csv](饭店数据/饭店推荐语义标注.csv)保存每条实体评论的原文、语义角色、是否有效推荐及判定依据，是实体汇总数字的可追溯明细；[饭店有效推荐评论.csv](饭店数据/饭店有效推荐评论.csv)只保留其中 1,468 条有效推荐，便于直接复核。
4. [饭店语义规则.csv](饭店数据/饭店语义规则.csv)保存经人工确认的视频级语境例外；规则限定到具体 BV，避免把其他视频里同名饭店的真实评价一并排除。
5. [门店核验.csv](饭店数据/门店核验.csv)保存具体分店、地址和平台链接。品牌命中但评论未指明分店时，不把不同分店强行合并。

两张实体生成表新增“涉及城市”和“涉及区县或县级市”字段，按 BV 号回连[视频来源.csv](饭店数据/视频来源.csv)。历史视频若已不在当前严格来源表中，这两个字段保持空白，不通过店名或文本猜测地区。

“评论提及数”只是包含该名称或别名的去重评论数；只有语义标注明确为“有效推荐”的评论才进入有效推荐计数。实体仍需完成分店和平台核验，不能只凭评论数进入 A／B／C 正式证据等级。

## 召回规则

- 词典命中：对已知正式名、常见简称、大小写和疑似错写进行统一，例如 `PUNCH`／`Punch House`、`鸡鸭鹅大饭店`／`无锡鸡鸭鹅大酒店`。
- 名称召回与推荐判断分离：正则和词典只用于发现可能的店名；推荐判断读取包含该实体的完整分句，不把整条评论中的正负词复制给所有店名。
- 比较／戏谑：`不如 X`、`还不如 X`、反问、表情和已人工确认的评论梗只算提及，不自动推导为对 X 的推荐。
- 有效推荐：需要出现针对该实体的明确到店／菜品建议、正向用餐体验、重复到店行为或处于明确推荐清单；负面、反讽和纯名称罗列单独计数。
- 门店后缀：召回以“面馆、饭店、食府、馄饨店、小笼、包子、烤肉、酸辣汤”等结尾的片段。
- 去重：先按 `BV号 + 评论ID` 去重评论，再按候选写法聚合；精筛表进一步按标准店名合并别名。
- 隐私最小化：输出不含评论者昵称、UID、头像或主页，只保留 BV 号、评论 ID 和最小必要证据摘录。

## 复跑

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File 饮食\脚本\提取饭店实体.ps1
```

默认读取 `%TEMP%\healthy_life_bili_comments\*.json`，并按[视频来源.csv](饭店数据/视频来源.csv)中的 BV 号过滤，只扫描本批正式来源。脚本会重建四张输出 CSV；别名判断应改在词典或候选人工审计表中，平台信息应改在门店核验表中，避免手工修改生成表后被覆盖。
